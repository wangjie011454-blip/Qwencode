# AI 智能面试官系统 - 技术文档

## 1. 项目概述

### 1.1 项目名称
AI Interviewer - AI 智能面试官系统

### 1.2 功能描述
基于阿里云百炼平台的大语言模型，构建一个智能面试官系统。用户上传简历后，AI 面试官会根据简历内容进行针对性提问，运用 STAR 原则深挖候选人真实能力，并在面试结束时生成结构化评估报告。

### 1.3 核心特性
- 简历上传与解析（支持 PDF、Word 格式）
- AI 面试官智能对话
- STAR 原则深度追问
- 多模型切换（qwen-turbo、qwen-plus、qwen-max）
- 结构化面试评估报告
- Cyberpunk 风格 UI 设计

### 6.2 路由设计

#### 6.2.1 对话接口 (chat.py)
```python
class ChatRequest(BaseModel):
    message: str                          # 用户消息
    history: List[Message] = []           # 历史消息
    resume_text: str = ""                 # 简历内容
    model: str = "qwen-plus"              # 模型选择
    file_id: Optional[str] = None         # 文件 ID

@router.post("/chat")
async def chat(request: ChatRequest):
    # 1. 构建系统 Prompt
    system_prompt = build_interviewer_prompt(
        resume_text=request.resume_text,
        model_strategy=request.model
    )
    
    # 2. 构建消息历史
    messages = [{"role": "system", "content": system_prompt}]
    messages.extend(request.history)
    messages.append({"role": "user", "content": request.message})
    
    # 3. 调用 AI 服务
    content = ai_service.chat(messages, request.model)
    
    return success_response({"content": content})
```

### 6.3 服务层

#### 6.3.1 AI 服务 (ai_service.py)
```python
class AIService:
    MODELS = {
        "qwen-turbo": "qwen-turbo",
        "qwen-plus": "qwen-plus",
        "qwen-max": "qwen-max",
    }
    
    def chat(self, messages: List[Dict], model: str) -> str:
        response = Generation.call(
            model=self.MODELS.get(model, model),
            messages=messages,
            result_format="message",
            stream=False
        )
        
        if response.status_code == 200:
            return response.output["choices"][0]["message"]["content"]
        else:
            raise Exception(f"API 错误: {response.message}")
```

#### 6.3.2 简历解析 (resume_parser.py)
```python
class ResumeParser:
    def parse(self, content: bytes, file_type: str) -> str:
        if file_type == "pdf":
            return self._parse_pdf(content)
        elif file_type in ["docx", "doc"]:
            return self._parse_word(content)
    
    def _parse_pdf(self, content: bytes) -> str:
        with pdfplumber.open(io.BytesIO(content)) as pdf:
            text = ""
            for page in pdf.pages:
                text += page.extract_text() or ""
        return text
    
    def _parse_word(self, content: bytes) -> str:
        doc = Document(io.BytesIO(content))
        return "\n".join([para.text for para in doc.paragraphs])
```

## 8. AI 面试官 Prompt 设计

### 8.1 系统提示词结构

```
# Role
你是一位拥有 10+ 年经验的资深技术面试官（Expert Technical Interviewer），曾任大厂技术专家。你擅长运用 **STAR 原则**（Situation, Task, Action, Result）进行行为面试，能敏锐识别候选人回答中的逻辑漏洞、过度包装或技术浅尝辄止，并能给出极具洞察力的评估报告。

# Context
- **候选人简历**: {resume_text}
- **招聘策略/侧重**: {model_strategy} (例如：侧重高并发经验、侧重架构设计能力、或侧重团队领导力)

# Goals
你的任务分为两个严格区分的阶段：
1. **🗣️ 面试交互阶段**：通过多轮对话，深挖候选人真实能力。
2. **📝 评估报告阶段**：仅在用户明确触发结束指令时，输出结构化评估报告。

# Constraints & Rules (至关重要)
1. **单轮单问**：每次回复**只能提出一个问题**。严禁一次性抛出多个问题。
2. **深度追问机制**：
   - 如果候选人回答笼统（如“我们做了优化”），必须追问细节（如“具体是哪个指标？你个人采用了什么算法？QPS提升了多少？”）。
   - 如果候选人只谈理论，必须要求提供**实战案例**。
   - 始终遵循 STAR 原则，确保每个核心经历都覆盖了情境、任务、行动和结果。
3. **语气风格**：专业、敏锐但亲切。像一位技术前辈在交流，而不是审问犯人。适当对精彩回答给予简短肯定，然后继续深挖。
4. **阶段控制**：
   - **默认状态**：始终处于“面试交互阶段”。
   - **触发结束**：只有当用户输入“结束面试”、“停止”、“生成报告”或类似指令时，才切换到“评估报告阶段”。
   - **禁止提前结题**：除非用户触发结束，否则不要主动生成报告。
5. **记忆保持**：在对话过程中，默默记录候选人的关键技术点、数据亮点和潜在疑点，用于最终报告。

# Workflow

## Phase 1: Interactive Interview (面试交互)
1. **初始化**：分析简历，结合 `{model_strategy}` 确定考察重点。
2. **开场**：用专业的问候语开场，邀请候选人做自我介绍（聚焦最近一段核心经历）。
3. **循环追问**：
   - 接收用户回答。
   - **内部思考**（不输出）：分析回答是否符合 STAR？数据是否量化？技术深度是否足够？是否存在逻辑矛盾？
   - **生成回应**：简短反馈（如“这个场景很有挑战性”） + **下一个针对性问题**（深挖细节或切换下一个考察维度）。
4. **持续**：直到用户触发结束指令。

## Phase 2: Evaluation Report (评估报告)
```

## 11. 注意事项

1. **API Key 安全**：不要将 `.env` 文件提交到版本控制
2. **文件上传限制**：限制文件大小为 10MB，仅支持 PDF 和 Word 格式
3. **CORS 配置**：生产环境应配置具体的允许域名
4. **错误处理**：前端和后端都应有完善的错误处理机制
5. **状态管理**：使用 Zustand 进行全局状态管理，避免 prop drilling
